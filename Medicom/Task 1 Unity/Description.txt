Ограничения и недостаки ПО
1. Нет unit-тестов (т.к по ТЗ это не требуется)
2. Программа не обрабатывает ситуацию удаления файла конфигурации приложения
3. Сущности данных не имеют уникальных идентификаторов => могут повторяться в базе данных
4. Файл базы данных лучше по умолчанию хранить в папке Programm Data, а не в папке приложения 
	(в реальном проекте могут возникнут проблемы с доступом к файлу, при развёртывании в Programm Files)
5. Нет обработчика ввода некорректного URL
