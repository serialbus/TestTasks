Ограничения и недостаки ПО
1. Программа не обрабатывает ситуацию удаления файла конфигурации приложения
2. Сущности данных не имеют уникальных идентификаторов => могут повторяться в базе данных
3. Файл базы данных лучше по умолчанию хранить в папке Programm Data, а не в папке приложения 
	(в реальном проекте могут возникнут проблемы с доступом к файлу, при развёртывании в Programm Files)
